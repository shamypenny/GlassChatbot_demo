# ATG研发端AI助手 Demo 启动指南

## 快速启动

### 方式一：一键启动（推荐）

#### Windows系统

**方法1：双击启动**
```
双击运行 demo/start_windows.bat
```

**方法2：命令行启动**
```powershell
cd demo
python run_demo.py
```

#### Linux/Mac系统

```bash
cd demo
chmod +x start_linux.sh
./start_linux.sh
```

---

### 方式二：手动启动

#### 步骤1：进入项目目录
```bash
cd demo
```

#### 步骤2：安装依赖
```bash
pip install -r requirements.txt
```

#### 步骤3：启动后端服务（终端1）
```bash
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
```

#### 步骤4：启动前端服务（终端2）
```bash
python -m streamlit run frontend/app.py --server.port 8501
```

## 配置说明

### API Key配置（可选）

如需使用真实的GLM大模型服务：

1. 复制环境变量模板：
```bash
cp .env.example .env
```

2. 编辑 `.env` 文件，填入智谱API Key：
```
ZHIPU_API_KEY=your-api-key-here
```

> **注意**：如不配置API Key，系统将使用模拟回答模式，仍可正常演示所有功能。

## 启动流程说明

### 一键启动流程

```
run_demo.py 执行流程：

1. 检查Python环境和依赖
   └── 自动安装缺失的依赖包

2. 初始化知识库
   └── 加载文献数据到Chroma向量库

3. 启动后端服务（端口8000）
   └── FastAPI + Uvicorn

4. 等待后端就绪（3秒）

5. 启动前端服务（端口8501）
   └── Streamlit

6. 显示访问地址
```

### 首次启动注意事项

首次启动时，系统会自动下载以下内容：

| 组件 | 大小 | 说明 |
|-----|------|------|
| BGE-M3 Embedding模型 | ~2GB | 中文语义向量模型 |
| 依赖包 | ~500MB | Python依赖库 |

**建议**：首次启动前确保网络稳定，下载完成后后续启动会更快。

## 常见问题

### Q1: 端口被占用怎么办？

**检查端口占用：**
```powershell
# Windows
netstat -ano | findstr :8000
netstat -ano | findstr :8501

# Linux/Mac
lsof -i :8000
lsof -i :8501
```

**解决方案：**
1. 结束占用进程
2. 或修改启动脚本中的端口号

### Q2: 依赖安装失败？

**尝试使用国内镜像：**
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q3: Embedding模型下载失败？

**手动下载模型：**
```python
from sentence_transformers import SentenceTransformer
model = SentenceTransformer('BAAI/bge-m3')
# 模型会自动下载到 ~/.cache/torch/sentence_transformers/
```

### Q4: 前端无法连接后端？

**检查清单：**
- [ ] 后端服务是否正常启动
- [ ] 访问 http://localhost:8000/docs 确认API可用
- [ ] 检查防火墙设置
- [ ] 确认前端代码中的 API_BASE 地址正确

## 服务管理

### 停止服务

- **一键启动方式**：按 `Ctrl+C` 停止
- **手动启动方式**：在各自终端按 `Ctrl+C` 停止

### 重启服务

```bash
# 停止所有Python进程（谨慎使用）
# Windows
taskkill /F /IM python.exe

# Linux/Mac
pkill -f python

# 重新启动
python run_demo.py
```

### 查看日志

后端日志会直接输出到终端，包含：
- API请求记录
- 错误信息
- 服务状态

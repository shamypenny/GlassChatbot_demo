# ATG研发端AI助手 Demo 部署指南

## 一、环境要求

### 硬件要求
- CPU: 4核及以上
- 内存: 8GB及以上（推荐16GB）
- 存储: 10GB可用空间
- GPU: 可选（用于加速Embedding）

### 软件要求
- Python: 3.9 - 3.11
- pip: 最新版本
- 操作系统: Windows 10/11, Ubuntu 20.04+, CentOS 7+

---

## 二、本地部署

### 方式一：一键启动（推荐）

**Windows:**
```bash
# 双击运行
start_windows.bat

# 或命令行运行
python run_demo.py
```

**Linux/Mac:**
```bash
chmod +x start_linux.sh
./start_linux.sh
```

### 方式二：手动启动

**步骤1：安装依赖**
```bash
cd demo
pip install -r requirements.txt
```

**步骤2：配置API Key（可选）**

复制环境变量模板：
```bash
cp .env.example .env
```

编辑 `.env` 文件，填入智谱API Key：
```
ZHIPU_API_KEY=your-api-key-here
```

> 如不配置API Key，系统将使用模拟回答模式

**步骤3：启动后端服务**
```bash
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
```

**步骤4：启动前端服务（新终端）**
```bash
python -m streamlit run frontend/app.py --server.port 8501
```

**步骤5：访问应用**
- 前端界面: http://localhost:8501
- API文档: http://localhost:8000/docs

## 三、服务器部署

### 方式一：Docker部署（推荐）

**步骤1：创建Dockerfile**
```dockerfile
FROM python:3.10-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000 8501

CMD ["python", "run_demo.py"]
```

**步骤2：构建镜像**
```bash
docker build -t atg-demo:latest .
```

**步骤3：运行容器**
```bash
docker run -d \
  --name atg-demo \
  -p 8000:8000 \
  -p 8501:8501 \
  -e ZHIPU_API_KEY=your-api-key \
  atg-demo:latest
```

### 方式二：系统服务部署

**步骤1：创建systemd服务文件**

后端服务 `/etc/systemd/system/atg-backend.service`:
```ini
[Unit]
Description=ATG Demo Backend
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/atg-demo
ExecStart=/usr/bin/python3 -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target
```

前端服务 `/etc/systemd/system/atg-frontend.service`:
```ini
[Unit]
Description=ATG Demo Frontend
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/atg-demo
ExecStart=/usr/bin/python3 -m streamlit run frontend/app.py --server.port 8501 --server.headless true
Restart=always

[Install]
WantedBy=multi-user.target
```

**步骤2：启动服务**
```bash
sudo systemctl daemon-reload
sudo systemctl enable atg-backend atg-frontend
sudo systemctl start atg-backend atg-frontend
```

### 方式三：Nginx反向代理

**Nginx配置示例**:
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端
    location / {
        proxy_pass http://127.0.0.1:8501;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }

    # 后端API
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # API文档
    location /docs {
        proxy_pass http://127.0.0.1:8000/docs;
    }
}
```

## 四、生产环境配置

### 环境变量配置

| 变量名 | 说明 | 默认值 |
|-------|------|-------|
| ZHIPU_API_KEY | 智谱API密钥 | 空（使用模拟模式） |
| ZHIPU_API_BASE | API地址 | https://open.bigmodel.cn/api/paas/v4 |
| EMBEDDING_MODEL | Embedding模型 | BAAI/bge-m3 |
| CHROMA_PERSIST_DIR | 向量库路径 | ./data/chroma_db |
| SQLITE_DB_PATH | 数据库路径 | ./data/glass.db |

### 性能优化建议

1. **Embedding模型加速**
   - 使用GPU加速
   - 或使用在线Embedding服务

2. **向量库优化**
   - 使用Milvus替代Chroma（大规模数据）
   - 配置合适的索引参数

3. **LLM调用优化**
   - 使用本地部署的GLM模型
   - 配置请求缓存

## 五、常见问题

### Q1: 启动失败，提示端口被占用
```bash
# 查看端口占用
netstat -ano | findstr :8000
netstat -ano | findstr :8501

# 结束占用进程
taskkill /PID <PID> /F
```

### Q2: Embedding模型下载失败
```bash
# 手动下载模型
from sentence_transformers import SentenceTransformer
model = SentenceTransformer('BAAI/bge-m3')
```

### Q3: API调用失败
- 检查API Key是否正确
- 检查网络连接
- 查看后端日志

### Q4: 前端无法连接后端
- 确认后端服务已启动
- 检查防火墙设置
- 确认API_BASE地址正确
